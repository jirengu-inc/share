大公司和小公司面试对比

准备过程

  1. 去年10月份开始准备一些面试知识点（GitHub，掘金，知乎等）
  2. 找一些公司来检验自己知识面的掌握程度

腾讯面试

  1. 第一面电话面试了2个小时，面试知识点主要是围绕js展开
    • js的垃圾回收机制
    • 异步的原理
    • http的相关知识
    • 。。。。
  2. 第二面就是业务和方向了，主要是问您平时在公司做过什么，项目经验等
  3. 第三面是算法面试。（泊松分酒，二维数组）
  4. 一个题目可以延伸多个知识点

YY面试

总部直接面试了一下午

  • 前端工程化
  • 前端性能优化
  • 前后端分离对于用户的好处
  • Promise A+规范
  • git的工作流
  • git rebase/cherry-pick

小公司面试

  1. 流程上可能不太一样（有时候莫名其妙的就收到offer 了）
  2. 可能围绕的点不一样，例如：vue/react的API，一些框架和库
  3. 可能还需要笔试
  4. 面试官的能力和挖掘能力

面试中的注意点

  1. 面试是双方的，可以适当的问问公司的情况
  2. 公司的开发流程
  3. 遇到不会的问题，可以直接说不了解，但是可以提出自己的观点
  4. 一定要在了解公司的工资构成后提出自己的期望工资

offer中的注意点

  1. 试用期和转正的区别
  2. 你的工资的基本构成
  3. 可以提前了解一些你的部门在公司的位置
  4. 五险一金的基数是多少（全额，最低标准）
